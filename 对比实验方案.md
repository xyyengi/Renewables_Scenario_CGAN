# CDM 与 CGAN 对比实验方案

## 1. 目标

本方案用于在德国周尺度电力场景生成任务上，对条件扩散模型（CDM）与条件生成对抗网络（CGAN）进行公平对比。对比目标不是模型内部损失，而是生成场景的质量、包络性和时序一致性。

本实验的核心原则是：

- 使用同一份数据
- 使用同一套数据划分
- 使用同一类条件输入
- 使用同一套训练/验证/测试规则
- 使用同一套后处理与评估指标

当前已经验证过的正式对比口径为：

- 测试集 25 周全部使用
- 每个测试周生成 100 个场景
- 先在 CDM 与 CGAN 两边都按这个口径跑一遍，再做最终对比

## 2. 对比对象

### 2.1 CDM

当前项目中的条件扩散模型，包含：

- [model_cdm.py](model_cdm.py)
- [diffusion_engine.py](diffusion_engine.py)
- [train_runner.py](train_runner.py)

### 2.2 CGAN

你另一个分支中的条件 GAN 模型。为了保证可比性，需要按本方案重新跑一遍，而不是沿用旧的全局训练结果。

## 3. 统一条件定义

这里的“条件”只有一个：`output_label`。

### 3.1 条件语义

`output_label` 表示周场景的出力等级，共三类：

- `0`：低出力
- `1`：中出力
- `2`：高出力

### 3.2 条件映射规则

在 CDM 中，条件会被映射成区间语义：

- `0 -> [0.0, 0.3]`，中点 `0.15`
- `1 -> [0.3, 0.7]`，中点 `0.50`
- `2 -> [0.7, 1.0]`，中点 `0.85`

这个映射在 [model_cdm.py](model_cdm.py) 中实现，并在 [diffusion_engine.py](diffusion_engine.py) 的采样阶段作为引导约束使用。

### 3.3 对 CGAN 的统一要求

CGAN 也必须使用同一份 `output_label` 作为条件输入。若 CGAN 原实现中只支持 one-hot 类别条件，则仍然使用这三个类别，但类别含义必须与上面的区间语义完全一致。

### 3.4 不能改变的条件内容

对比时必须固定以下内容，不允许一边改、一边不改：

- 条件变量本身：`output_label`
- 条件类别数：3 类
- 条件语义：低 / 中 / 高出力
- 条件和真实测试周的对应关系

## 4. 数据集与划分

### 4.1 数据集

- 输入文件：`germany_data_final.csv`
- 特征维度：504
- 结构：168 小时负荷 + 168 小时风电 + 168 小时光伏
- 标签：`output_label`

### 4.2 必须采用的划分

总样本数 165 周，固定为：

- 训练集：115
- 验证集：25
- 测试集：25

### 4.3 划分要求

CDM 和 CGAN 必须使用同一组索引划分，不允许各自随机切分，否则对比无效。

建议做法：

- 固定数据顺序
- 固定 `train/valid/test` 索引
- 两个模型都直接读取同一份索引文件或同一套代码逻辑

## 5. 必须控制的单一变量

下面这些变量在对比时必须保持一致，否则实验不公平。

### 5.1 数据相关

- 同一份 `germany_data_final.csv`
- 同一套 115/25/25 划分
- 同一个 `germany_scaler.pkl`
- 同样的逆归一化流程

### 5.2 条件相关

- 同一个 `output_label`
- 同一个三分类定义
- 同一个低/中/高语义
- 同一个条件分布，不额外加入新条件

### 5.3 生成相关

- 同样的测试周集合：25 个测试周全部使用
- 同样的每周生成场景数：100 个
- 同样的随机种子策略
- 同样的输出维度 504
- 同样的后处理方式

### 5.4 评估相关

- 同样在 MW 尺度上做评估
- 同样计算 ES / CR / SW / ACF
- 同样的 ACF 最大滞后阶数
- 同样的 coverage alpha

## 6. 实验流程

### 6.1 CDM 流程

1. 读取 `germany_data_final.csv`
2. 按固定索引划分 115/25/25
3. 用训练集拟合或加载 `germany_scaler.pkl`
4. 在归一化空间训练 CDM
5. 在全部 25 个测试周上，按 `output_label` 条件生成每周 100 个场景
6. 逆归一化回 MW
7. 计算 ES / CR / SW / ACF

### 6.2 CGAN 流程

1. 读取同一份 `germany_data_final.csv`
2. 按同一组索引划分 115/25/25
3. 使用同一个 `germany_scaler.pkl`
4. 按同一个 `output_label` 条件训练 CGAN
5. 在全部 25 个测试周上，按相同条件生成每周 100 个场景
6. 逆归一化回 MW
7. 计算 ES / CR / SW / ACF

## 7. 评估指标

评估必须在生成场景之后进行，不直接看训练 loss。

### 7.1 Energy Score (ES)

衡量生成场景集与真实观测的概率一致性。值越小越好。

### 7.2 Coverage Rate (CR)

衡量真实值落入场景包络区间的比例。值越高越好。

### 7.3 Scenario Width (SW)

衡量场景分布带宽。越大说明场景越分散，但过大可能意味着过度发散。

### 7.4 ACF

衡量生成场景与真实数据在 168 小时内的自相关结构是否一致。重点看曲线形状是否接近。

## 8. 对比时应该看什么

### 8.1 主结论指标

建议优先比较：

- 总体 ES
- 总体 CR
- 总体 SW
- 总体 ACF 曲线

### 8.1.1 当前正式对比口径

建议把下面这套作为 CDM 和 CGAN 的正式对比口径：

- 测试周数：25
- 每周生成场景数：100
- 条件：`output_label` 三分类低/中/高出力
- scaler：`germany_scaler.pkl`
- 评估尺度：逆归一化后的 MW

如果 CPU 资源非常紧张，可以先用更小的快速验收口径做流程检查，但正式汇报仍建议回到上面的 25 周 / 100 场景口径。

### 8.2 分组指标

建议进一步按条件分组比较：

- 低出力组（label 0）
- 中出力组（label 1）
- 高出力组（label 2）

这样可以判断模型是否在某一类场景上明显失真。

### 8.3 视觉检查

建议额外抽样看：

- 每个 label 下的典型生成周曲线
- 真实曲线与生成中位数曲线
- 负荷 / 风电 / 光伏三部分是否保持同步波动

## 9. 公平性检查清单

重跑 CGAN 前，先检查下面这些点是否全部一致：

- [ ] 数据集相同
- [ ] 划分索引相同
- [ ] scaler 相同
- [ ] 条件定义相同
- [ ] 条件类别数相同
- [ ] 输出维度相同
- [ ] 测试集相同
- [ ] 每个测试周的条件输入相同
- [ ] 生成场景数相同（正式口径为 100）
- [ ] 评估指标相同
- [ ] 逆归一化后再评估

## 10. 推荐的汇报方式

最终结果建议整理成三层：

1. 总体对比表：CDM vs CGAN 的 ES / CR / SW / ACF
2. 按 label 分组的对比表：0 / 1 / 2 三类分别比较
3. 典型样本图：每个 label 画 1 到 3 个测试周

## 11. 结论口径建议

如果后续写报告或论文，建议把结论表述为：

- 在统一的条件定义下，CDM 和 CGAN 生成同一批测试周场景
- 在同一数据划分和同一 scaler 下进行逆归一化和指标计算
- 用 ES / CR / SW / ACF 比较两种模型的概率一致性、覆盖能力和时序结构保留能力

这样结论会更稳，也更容易说明实验公平。
